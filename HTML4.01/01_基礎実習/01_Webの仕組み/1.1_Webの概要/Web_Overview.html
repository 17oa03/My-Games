1.1_Webの概要

▼Web の基本的な概念は、文書形式である HTML(HyperText Markup Language)、文書転送プロトコルである HTTP(HyperText Transter Protocol)、文書の所在の指定方法である URI(Uniform Resource Identifier)の3つからなる。

▼URI は所謂「アドレス」のことであり、Web 上のデータの所在地を示すものである。URI は以前は「URL(Uniform Resource Locator)」と呼ばれていたが、現在では広い概念として URI という用語が使われることが多い。

▼HTTP は Web サーバーとクライアント(ブラウザなど)がデータを送受信する際に準拠する通信規約(プロトコル)のことであり、Web ページは基本的にこの HTTP に基づいてやり取りされる。現在では HTTP/1.0 と HTTP/1.1 の二つのバージョンが一般に利用されている。

▼つまり、Web 上に存在する HTML で作成された文書(Webページ)について、ブラウザでその文書の URI を指定することで、HTTPに従ってその文書のデータが転送され画面に表示される、という流れになる。

  1. Web クライアントから URI を指定し、データを Web サーバーに要求する。
  2. Web サーバーから要求に応じて HTML を転送する。
  3. Web サーバーから転送された HTML を Web クライアントのブラウザで表示する。
