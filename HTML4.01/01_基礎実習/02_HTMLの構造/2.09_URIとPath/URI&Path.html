2.9_URIとPath

▼URIは次のような構造になっている。

http://(スキーム) www.w3.org/(サーバー名) TR/html4/strict.dtd(パス＋ファイル名) 

1)スキーム
    これによってブラウザはリソースにアクセスする方法(プロトコル)を判断する。代表的なスキームとしては「http:」「https:」「ftp:」などがある。

2)サーバー名(ドメイン名)
    リソースを取得する為に、アクセスするコンピューターの名前を示す。

3)パス＋ファイル名
    アクセスするコンピューター内のリソースの所在を表す。

▼リンクを指定したり画像を表示させたりする為に、リンク先の HTML ファイルや画像ファイルの位置を正確に記述する必要がある。この記述方法には 絶対パス と 相対パス の2通りの方法がある。 

4)絶対パス
    あるファイルの位置を全体から見て、一番元になる位置から階層構造を順番に辿って記述する方法で「http://」から始まる。主に他のサイトにあるページや画像を指定する時に使用する。

5)相対パス
    ファイル同士の位置を基準とするファイルから見て、どこにあるか「相対的に」記述する方法。自分のWebサイト内では主にこちらの方法を使用する。現在より1つ上の階層のディレクトリは「../」で表す。
